# 画面設計：間違いノートアプリ

## 画面遷移図

```
                    ┌─────────────────────────────────────────┐
                    │          左サイドバーナビゲーション          │
                    │  ホーム ／ 苦手問題一覧 ／ 科目・単元管理     │
                    └─────────────────────────────────────────┘

ホーム
（今日の復習一覧）
    │
    ├─ 問題をタップ ──→ 復習
    │                    │
    │                    ├─ 正解 ──→ ホームに戻る
    │                    │
    │                    └─ 不正解 ──→ メモ・復習日を編集 ──→ ホームに戻る
    │
    └─ 「＋」ボタン ──→ 問題登録 ──→ ホームに戻る

苦手問題一覧
    │
    └─ 「復習する」ボタン ──→ 復習（同上）

克服済み一覧（苦手問題一覧内のタブで切り替え）
```

---

## 各画面詳細

### ホーム（今日の復習一覧）

**目的：** `next_review_at` が今日以前のアクティブな問題を一覧表示し、復習を開始する。

**表示データ**
- 問題文（先頭50文字程度）
- 科目名 / 単元名
- 間違えた回数（wrong_count）
- 次の復習日（next_review_at）

**操作**
- 問題をタップ → 復習へ
- 「＋」ボタン → 問題登録へ

**使用API**
- `GET /mistake-notes/today`

---

### 問題登録

**目的：** 問題文・正解・間違えた理由・次の復習日を一画面で登録する。

**レイアウト：** 2カラム（左：問題・正解 / 右：メモ・復習日）

**表示・入力**
- 科目（セレクトボックス）
- 単元（科目選択後に動的に表示、任意）
- 問題文（テキストエリア）
- 正解（テキストエリア）
- 間違えた理由（テキストエリア、任意）
- 今回学んだこと（テキストエリア、任意）
- 次の復習日（日付ピッカー、任意）

**操作**
- 登録ボタン → 問題を保存してホームに戻る
  - メモまたは復習日が入力された場合、`mistake_notes` も自動作成

**使用API**
- `GET /subjects`
- `GET /subjects/{subject_id}/units`
- `POST /questions`（`memo`・`learning`・`next_review_at` を同時送信）

---

### 復習

**目的：** 問題を解いて正誤を記録し、不正解の場合はメモ・復習日を更新する。

**表示・操作の流れ**
1. 問題文を表示（正解は非表示）
2. 「答えを見る」→ 正解を表示
3. 正解 / 不正解 を選択
   - 正解 → ホームに戻る
   - 不正解 → メモ編集フォームを展開
4. メモ・復習日を入力して保存 → ホームに戻る

**表示データ**
- 問題文（question_text）
- 科目名 / 単元名
- 間違えた回数（wrong_count）
- 「答えを見る」押下後に表示：
  - 正解（correct_answer）
  - 間違えた理由（memo）※未設定の場合は非表示
  - 今回学んだこと（learning）※未設定の場合は非表示
- 不正解時に編集可能：
  - 間違えた理由（memo）
  - 今回学んだこと（learning）
  - 次の復習日（next_review_at）

**使用API**
- `POST /questions/{id}/attempts`（正解/不正解を記録）
- `PUT /mistake-notes/{id}`（不正解時：メモ・復習日を更新）

---

### 苦手問題一覧

**目的：** active な mistake_notes を一覧表示する。克服済み一覧はタブで切り替え。

**タブ**
- 「苦手中」（active） / 「克服済み」（mastered）

**表示データ（苦手中タブ）**
- 問題文（先頭50文字程度）
- 科目名 / 単元名
- 間違えた回数
- 次の復習日

**表示データ（克服済みタブ）**
- 問題文
- 科目名 / 単元名
- 間違えた回数

**操作（苦手中タブ）**
- 「復習する」ボタン → 復習へ
- 「克服済みにする」ボタン → その場で mastered に変更

**使用API**
- `GET /mistake-notes`（苦手中タブ）
- `GET /mistake-notes/mastered`（克服済みタブ）
- `PUT /mistake-notes/{id}/mastered`

---

### 科目・単元管理

**目的：** 科目・単元の追加・編集・削除を行う。

**表示**
- 科目一覧
  - 各科目の下に単元一覧を折りたたみ表示

**操作**
- 科目を追加（テキスト入力 → 保存）
- 科目名を編集（インライン編集）
- 科目を削除（紐づく問題がある場合はエラーメッセージ表示）
- 単元を追加（科目ごとにテキスト入力 → 保存）
- 単元名を編集（インライン編集）
- 単元を削除（紐づく問題がある場合はエラーメッセージ表示）

**使用API**
- `GET /subjects`
- `POST /subjects` / `PUT /subjects/{id}` / `DELETE /subjects/{id}`
- `GET /subjects/{subject_id}/units`
- `POST /subjects/{subject_id}/units` / `PUT /units/{id}` / `DELETE /units/{id}`
