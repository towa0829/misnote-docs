# 画面設計：間違いノートアプリ

## 画面遷移図

```
                    ┌─────────────────────────────────────────┐
                    │          左サイドバーナビゲーション          │
                    │  ホーム ／ 苦手問題一覧 ／ 科目・単元管理     │
                    └─────────────────────────────────────────┘

ホーム
（今日の復習一覧＋復習日未設定）
    │
    ├─ 問題をタップ ──→ 復習
    │                    │
    │                    ├─ 正解 ──→ 次の復習日を設定 ──→ ホームに戻る
    │                    │            ├─ 3連続正解なら「克服済みにしますか？」と提案
    │                    │            └─ 克服済みにする ──→ ホームに戻る
    │                    │
    │                    └─ 不正解 ──→ メモ・復習日を編集 ──→ ホームに戻る
    │
    └─ 「＋」ボタン ──→ 問題登録 ──→ ホームに戻る

苦手問題一覧
    │
    └─ 「復習する」ボタン ──→ 復習（同上）

克服済み一覧（苦手問題一覧内のタブで切り替え）
    │
    └─ 「苦手に戻す」ボタン ──→ active に戻す
```

## 各画面詳細

### ホーム（今日の復習一覧）

**目的：** `next_review_at` が今日以前のアクティブな問題を一覧表示し、復習を開始する。

**表示データ**
- 問題文（先頭50文字程度）
- 科目名 / 単元名
- 間違えた回数（wrong_count）
- 次の復習日（next_review_at）

**復習日未設定セクション**
- active かつ `next_review_at` が `null` のノートを「復習日未設定」として今日の復習の下に表示する
- `GET /mistake-notes` の結果をクライアント側でフィルタして表示し、復習日の設定を促す
- 復習日を設定しない限り「今日の復習」には出ないため、放置による忘却を防ぐ目的

**操作**
- 問題をタップ → 復習へ
- 「＋」ボタン → 問題登録へ
- 復習日未設定の問題 → 復習日を設定（日付ピッカー → `PUT /mistake-notes/{id}`）

**使用API**
- `GET /mistake-notes/today`
- `GET /mistake-notes`（復習日未設定セクション用）
- `PUT /mistake-notes/{id}`（復習日の設定）

---

### 問題登録

**目的：** 問題文・正解・間違えた理由・次の復習日を一画面で登録する。

**レイアウト：** 2カラム（左：問題・正解 / 右：メモ・復習日）

**表示・入力**
- 科目（セレクトボックス）
- 単元（科目選択後に動的に表示、任意）
- 問題文（テキストエリア）
- 正解（テキストエリア）
- 間違えた理由（テキストエリア、任意）
- 今回学んだこと（テキストエリア、任意）
- 次の復習日（日付ピッカー、任意）

**操作**
- 登録ボタン → 問題を保存してホームに戻る
  - メモまたは復習日が入力された場合、`mistake_notes` も自動作成

**使用API**
- `GET /subjects`
- `GET /subjects/{subject_id}/units`
- `POST /questions`（`memo`・`learning`・`next_review_at` を同時送信）

---

### 復習

**目的：** 問題を解いて正誤を記録し、正解時は次の復習日の設定・克服済み化、不正解時はメモ・復習日の更新を行う。

**表示・操作の流れ**
1. 問題文を表示（正解は非表示）
2. 自分の答えを入力（テキストエリア、**任意**。書かずに次へ進んでもよい）
3. 「答えを見る」→ 正解を表示
4. 正解 / 不正解 を選択（自己採点。attempt として記録される）
   - 正解 → 次回設定フォームを展開
     - レスポンスの `mastery_suggested` が true の場合、「3回連続で正解しています。克服済みにしますか？」の提案を表示（克服ボタンを強調）
     - 次の復習日を設定して保存 → ホームに戻る
     - または「克服済みにする」→ その場で mastered に変更 → ホームに戻る（提案の有無にかかわらず手動で克服できる）
   - 不正解 → メモ編集フォームを展開
5. メモ・復習日を入力して保存 → ホームに戻る

**表示データ**
- 問題文（question_text）
- 科目名 / 単元名
- 間違えた回数（wrong_count）／連続正解回数（correct_streak）
- 「答えを見る」押下後に表示：
  - 正解（correct_answer）
  - 間違えた理由（memo）※未設定の場合は非表示
  - 今回学んだこと（learning）※未設定の場合は非表示
- 正解時に設定可能：
  - 次の復習日（next_review_at）
  - 「克服済みにする」ボタン → mastered に変更（`mastery_suggested` 時は提案バナーで強調）
- 不正解時に編集可能：
  - 間違えた理由（memo）
  - 今回学んだこと（learning）
  - 次の復習日（next_review_at）

**使用API**
- `GET /mistake-notes/{id}`（URL直接アクセス・リロード時のデータ取得）
- `POST /questions/{id}/attempts`（正解/不正解と任意の user_answer を記録。レスポンスに correct_streak / mastery_suggested）
- `PUT /mistake-notes/{id}`（メモ・復習日を更新）
- `PUT /mistake-notes/{id}/status`（正解時：「克服済みにする」で mastered 化）

---

### 苦手問題一覧

**目的：** active な mistake_notes を一覧表示する。克服済み一覧はタブで切り替え。

**タブ**
- 「苦手中」（active） / 「克服済み」（mastered）

**表示データ（苦手中タブ）**
- 問題文（先頭50文字程度）
- 科目名 / 単元名
- 間違えた回数 / 連続正解回数（例：「連続正解 2回」）
- 次の復習日

**表示データ（克服済みタブ）**
- 問題文
- 科目名 / 単元名
- 間違えた回数

**操作（苦手中タブ）**
- 「復習する」ボタン → 復習へ
- 「克服済みにする」ボタン → その場で mastered に変更

**操作（克服済みタブ）**
- 「苦手に戻す」ボタン → active に戻す（復習日は未設定に戻るため、ホームの「復習日未設定」から再設定する）

**使用API**
- `GET /mistake-notes`（苦手中タブ）
- `GET /mistake-notes/mastered`（克服済みタブ）
- `PUT /mistake-notes/{id}/status`（克服済みにする / 苦手に戻す）

---

### 科目・単元管理

**目的：** 科目・単元の追加・編集・削除を行う。

**表示**
- 科目一覧
  - 各科目の下に単元一覧を折りたたみ表示

**操作**
- 科目を追加（テキスト入力 → 保存）
- 科目名を編集（インライン編集）
- 科目を削除（紐づく問題がある場合はエラーメッセージ表示）
- 単元を追加（科目ごとにテキスト入力 → 保存）
- 単元名を編集（インライン編集）
- 単元を削除（紐づく問題がある場合はエラーメッセージ表示）

**使用API**
- `GET /subjects`
- `POST /subjects` / `PUT /subjects/{id}` / `DELETE /subjects/{id}`
- `GET /subjects/{subject_id}/units`
- `POST /subjects/{subject_id}/units` / `PUT /units/{id}` / `DELETE /units/{id}`

## 共通UI仕様

全画面に共通するUI/UXのルール。個別画面の記述より優先される。

### カラー・アクセシビリティ

- カラートークン（モックアップの `:root` が正）:
  - `--amber: #B45309` — 主要ボタン・リンク（白文字/白地で 5.0:1 を確保）
  - `--amber-dk: #92400E` — ホバー時
  - `--amber-br: #D97706` — navy サイドバー上のアクセント専用（ロゴ・アクティブ表示・プログレスバー）。**白背景上のテキスト・ボタンには使わない**（3.2:1 でWCAG AA未達）
- テキストと背景のコントラストは **4.5:1 以上**（WCAG AA）。色だけで意味を伝えず、アイコンまたはテキストを併記する（例: 間違い回数は「×」アイコン＋赤）
- すべてのインタラクティブ要素に可視フォーカスリング（`:focus-visible`、2px アウトライン）
- アイコンのみのボタンには `aria-label` を必須とする

### インタラクション

- クリック/タップ対象は最小 **32×32px**（モバイルでは 44×44px）。小さいアイコンはパディングで当たり判定を広げる
- ボタンは押下中 disabled にし、スピナーを表示（二重送信防止）。300ms を超えるロードにはスケルトンまたはスピナー
- 保存・登録成功時はトーストで通知（3〜5秒で自動消滅、`aria-live="polite"`）
- **削除は確認ダイアログを挟む**（科目・単元・問題）。紐づくデータがあり削除できない場合（409）は、理由と対処（「先に問題を移動または削除してください」）をダイアログ内に表示
- フォームエラーは対象フィールドの直下に赤字＋アイコンで表示し、最初のエラーフィールドにフォーカスを移す
- 次の復習日の日付ピッカーは `min` を今日に設定（過去日は選択不可）

### 空状態（Empty States）

| 画面 | 条件 | 表示 |
|------|------|------|
| ホーム | 今日の復習が0件 | 「今日の復習は完了！」の達成メッセージ＋「問題を登録する」ボタン |
| ホーム | ノートが1件もない | 「間違えた問題を登録して復習をはじめよう」＋登録ボタン |
| 苦手問題一覧（苦手中） | active が0件 | 「苦手問題はありません」＋登録への導線 |
| 苦手問題一覧（克服済み） | mastered が0件 | 「克服済みの問題はまだありません。復習を続けよう」 |
| 科目・単元管理 | 科目が0件 | 「まず科目を追加しましょう（例: 数学・英語）」＋追加ボタン |

### レスポンシブ方針

ターゲット（小中高生）はスマートフォン利用が主と想定し、実装はモバイルファーストで行う。

- ブレークポイント: 375px（スマホ基準）/ 768px / 1024px / 1440px
- **1024px以上**: 現行モックアップどおり左サイドバーナビゲーション
- **1024px未満**: サイドバーを下部ナビゲーション（ホーム / 苦手問題 / 科目管理 の3項目、アイコン＋ラベル）に置き換え。「＋問題を登録」はヘッダー右上に維持
- 問題登録の2カラムレイアウトは 768px 未満で1カラムに積む
- 苦手問題一覧のテーブルは 768px 未満でカード型リストに切り替え
- 本文フォントは 16px 以上（iOS の自動ズーム回避）、横スクロールは発生させない
- `prefers-reduced-motion` 時はアニメーションを無効化する

> 現行の HTML モックアップはデスクトップ（1280px）のリファレンス。モバイルレイアウトは上記方針に従って実装時に展開する。
